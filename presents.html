Introduction

Initial motivation came from John Rusnak to observe if the interferences caused any data anomaly issues.

The goal was to watch if the algorithm performed reasonably well in a scenario where the sensors would be in use.

Wi-Fi Interference

  • Bombard mode
  • Time interval
  • Variabe packet size
  • Variable time interval

GPS Interference

This mode just updates and searches for the user location every 1 second.

The location is retrieved every second assuming that the user is stationary.

CPU Stress Interference

This mode spawns background processes based on the duty cycle.

Android system limits the value of duty cycle to 50%.

Tethering

Just a small Ubuntu native application to display the sensor values from a Beagle board/tab.

Improvements

Though the Android app and the tethering app are well contained and robust, there is a reasonable scope for improvement.

Improving the file upload capability from the device to the server is of prime importance.

Displaying the results on the android device from the server in a responsive way is another feature to be improved.
